# Plan — составить план реализации

## Шаг 1 — Первый раунд уточнений

Задай 4 вопроса через AskUserQuestion. Покрой эти оси:

- **Масштаб:** минимальный рабочий результат vs полная реализация
- **Конечный результат:** что именно должно существовать (файл, функция, сервис, APK, документ)
- **Ограничения:** что нельзя трогать, какой стек зафиксирован
- **Приоритет:** скорость / качество кода / демо-готовность

## Шаг 2 — Второй раунд уточнений

На основе ответов задай ещё 2–3 вопроса — уже конкретные:

- Интеграция с каким компонентом (Android-виджет / NLP / mock API)
- Кто будет использовать результат (Яна, Денис, жюри)
- Нужны ли тесты или достаточно рабочего примера

## Шаг 3 — Составить план

- Разбей на шаги (не больше 7), каждый — одно действие с понятным результатом
- Укажи зависимости между шагами
- Перечисли файлы, которые будут созданы или изменены
- Отдельно — что явно НЕ входит в план (scope boundary)

## Шаг 4 — Подтверждение

Спроси через AskUserQuestion: приступать или скорректировать?
Варианты: «Приступай», «Скорректировать», «Только план, не выполнять».
