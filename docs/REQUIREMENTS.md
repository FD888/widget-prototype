# Требования к прототипу VTB Vita

---

## Функциональные требования (MVP Phase 1)

### FR-01 — Виджет на главном экране
- Виджет добавляется на главный экран Android стандартным способом (AppWidget)
- Занимает строку во весь экран (4×1 или 4×2 ячейки)
- Содержит: кнопку голосового ввода, поле для ввода текстовой команды, поверх него находится полупрозрачный текст с контекстной надписью, при нажатии, текст пропадает, появляется клавиатура, можно набрать запрос

### FR-02 — Ввод команды
- Поддерживается текстовый ввод (обязательно для Phase 1)
- Голосовой ввод — Phase 2 (C-08), для Phase 1 достаточно текста

### FR-03 — NLP-распознавание
- Распознаёт 3 intent'а: `transfer`, `balance`, `topup`
- Для `transfer` — извлекает имя получателя и сумму
- Для `topup` — определяет номер телефона (свой по умолчанию)
- Поиск получателя по mock-истории переводов (нечёткое совпадение по имени)

### FR-04 — Модал подтверждения
- Показывает: получатель, сумма, счёт списания, баланс после
- Кнопки «Отправить» и «Отмена»
- Заглушка биометрии — кнопка «Подтвердить» (без реального BiometricPrompt)

### FR-05 — Статус операции
- После подтверждения — показывает результат в виджете или Toast
- Мок-ответы: успех / ошибка

### FR-06 — Mock-данные
- История переводов: минимум 5 контактов (разные имена, карты)
- Баланс: захардкожен или из JSON
- Все данные вымышленные

---

## Нефункциональные требования

### NFR-01 — Платформа
- Android минимум API 26 (Android 8.0)
- Целевой API: 34 (Android 14)

### NFR-02 — Время отклика
- Команда → модал: не более 3 секунд (включая обращение к NLP-сервису)
- Если NLP-сервис недоступен — graceful fallback на mock-данные

### NFR-03 — Публичный доступ (для C-05)
- APK доступен для скачивания по публичной ссылке, ИЛИ
- Есть инструкция по сборке из исходников (README)

### NFR-04 — Воспроизводимость
- `README.md` содержит инструкцию запуска NLP-сервиса за 3 команды или меньше
- Android-приложение собирается командой `./gradlew assembleDebug`

---

## Out of scope (не реализуем в прототипе)

- Реальные банковские API и реальные транзакции
- Настоящая биометрия (BiometricPrompt)
- JWT и аутентификация
- iOS-версия
- Пуш-уведомления (FCM)
- Сложные операции: инвестиции, кредиты, конвертация
- Поддержка нескольких счетов (только «основной»)
- Отмена транзакции после подтверждения

---

## Критерии приёмки Phase 1 (C-05 закрыт когда)

- [ ] Виджет устанавливается на реальный Android-телефон
- [ ] Текстовая команда «переведи Кате 500» → модал с Катей и суммой 500
- [ ] Кнопка «Отправить» → статус «Деньги отправлены»
- [ ] Команда «покажи баланс» → баланс в виджете или модале
- [ ] NLP-сервис доступен по публичному URL
- [ ] README позволяет воспроизвести демо с нуля
