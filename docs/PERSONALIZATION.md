# VTB Vita — Персонализация виджета

> Статус: **Нейтральные** ✅ · **Напоминания** ✅ · **ВЫГОДА** ✅ · **Аналитика** 📋 спроектирована  
> Последнее обновление: 23 апреля 2026

---

## Концепция

Виджет показывает персонализированные подсказки — короткие фразы в капсуле и баннер в InputActivity.  
Подсказки делятся на четыре типа:

| Тип | Цвет на виджете | Баннер в InputActivity | Реализован |
|-----|----------------|------------------------|-----------|
| **Нейтральный** | приглушённый (стандартный) | нет | ✅ |
| **Напоминание** | янтарный | ✅ НАПОМИНАНИЕ | ✅ |
| **ВЫГОДА** | зелёный | ✅ ВЫГОДА | ✅ |
| **Аналитика** | синий (ВТБ) | нет | 📋 |

**Ключевой принцип:** виджет не создаёт информационное давление. Большую часть времени он просто есть — нейтрально и ненавязчиво.

**Тест фразы:** звучит ли это как совет от друга-финансиста? Если звучит как менеджер банка — выбросить.

## Тип 0 — Нейтральные (ambient) ✅

Фразы без информационного посыла. Виджет живой, не давит.  
Показываются по умолчанию когда ни один другой триггер не сработал.

**Реализация:** `NeutralHints.kt` — локальный пул, офлайн, без API.

**Правила ротации:**
- Меняется после **5 открытий** или **3 часов** (что наступит раньше)
- Без повторов два раза подряд
- Счётчик открытий: `incrementOpenCount()` в `hideWidget()`

**Пул по времени суток:**

| Период | Фразы |
|--------|-------|
| 6–10 | Доброе утро · С добрым утром · Хорошего утра |
| 11–16 | На связи · Слушаю · Готов помочь · Всё под контролем · Чем помочь? · Как дела? |
| 17–21 | Добрый вечер · Как прошёл день? · На связи · Всё в порядке? |
| 22–5 | На связи · Тихо сегодня · Всё под контролем |

**Праздники и особые дни (приоритет выше времени суток):**

| Дата | Фразы | Аудитория |
|------|-------|-----------|
| 23 февраля | С Днём защитника · С праздником | мужчины |
| 8 марта | С 8 Марта · С весенним праздником | женщины |
| 31 декабря | С наступающим · Скоро Новый год | все |
| 1 января | С Новым годом · Счастливого Нового года | все |
| 9 мая | С Днём Победы · С праздником | все |
| 12 июня | С Днём России | все |
| 1 сентября | С новым учебным годом · Первое сентября | все |
| День рождения (сегодня) | С днём рождения · Сегодня твой день | персонально |
| День рождения (завтра) | Завтра день рождения | персонально |

Пол берётся из профиля персоны (`gender: "male" | "female"`), дата рождения — `birthday: "MM-dd"`.

## Тип 1 — Напоминания ✅

Своевременные, конкретные, полезные. Только когда есть реальное событие — это сервис, не реклама.

### Источник данных

Таблица `scheduled_payments` в SQLite, поле `payment_type`:

| `payment_type` | Примеры | Логика напоминания |
|---------------|---------|-------------------|
| `credit_card` | Кредитная карта ВТБ | за 3 дня + **день-в-день** если не оплачено |
| `loan` | Ипотека ВТБ | за 3 дня + **день-в-день** если не оплачено |
| `autopayment` | ТТК Интернет, ЖКХ | только за 2–3 дня, без day-of (нет рисков просрочки) |
| `subscription` | Netflix, Яндекс.Плюс | **не показывается** в напоминаниях |

### Окно напоминания и текст

| days_until_due | Тип | Текст виджета | Срочность |
|---------------|-----|--------------|-----------|
| < 0 (просрочен) | credit_card / loan | `{Имя} — просрочен · {сумма} ₽` | urgent |
| 0 (сегодня) | credit_card / loan | `{Имя} — сегодня последний день · {сумма} ₽` | urgent |
| 1 | любой | `{Имя} — завтра · {сумма} ₽` | upcoming |
| 2–3 | любой | `{Имя} — через {N} дня · {сумма} ₽` | upcoming |
| > 3 | — | не показывается | — |

### Приоритет при нескольких платежах

```
1. Просроченный credit_card / loan (urgency_order=0)
2. Сегодняшний credit_card / loan (urgency_order=0)
3. Ближайший credit_card / loan в окне (urgency_order=1)
4. Ближайший autopayment в окне (urgency_order=2)
```

Среди одного уровня — выбирается с наименьшим `days_until_due`.

### Баннер в InputActivity

```
┌─ НАПОМИНАНИЕ ──────────────────────────────────────────┐
│  ТТК Интернет — через 2 дня · 690 ₽      Оплатить →   │
└────────────────────────────────────────────────────────┘
```

Тап → `requirePin` → `/command pay_scheduled` → `ConfirmActivity` → оплата.

### Данные для демо

| Персона | Платёж | Тип | День месяца | Сумма |
|---------|--------|-----|------------|-------|
| Ольга | ТТК Интернет | autopayment | 24-е | 690 ₽ |
| Ольга | ЖКХ — УК Центральная | autopayment | 25-е | 6 800 ₽ |
| Ольга | Ипотека ВТБ | loan | 28-е (вне окна) | 38 500 ₽ |
| Артём | Кредитная карта ВТБ | credit_card | 24-е | 2 700 ₽ |
| Витя | — | — | — | нет напоминания → VygodaBanner |

## Тип 2 — ВЫГОДА ✅

Формат: тег **`ВЫГОДА`** (зелёный) + **числовая выгода** + **1 действие**.  
Без тройки `тег + цифра + действие` — не показывать.

Реализация: `GET /hint` на сервере → выбор оффера через `db.get_vygoda_offer()` с учётом `active_products`, баланса, сегмента, возраста, трат.

### Полный пул продуктов

#### Cashback / карты
| ID | Триггер | Текст баннера | CTA | Действие |
|----|---------|---------------|-----|---------|
| `cashback_cafe` | топ-категория трат = кафе/рестораны | `Кафе — топ расходов · кэшбэк вернёт ~{N} ₽/мес` | Подключить → | ProductsTab |
| `cashback_grocery` | топ-категория трат = продукты | `Продукты: {X} ₽/мес · подключи кэшбэк → +{N} ₽ назад` | Подключить → | ProductsTab |
| `cashback_transport` | топ-категория трат = транспорт | `Транспорт: {X} ₽/мес · кэшбэк вернёт ~{N} ₽` | Подключить → | ProductsTab |
| `cashback_multi` | нет кэшбэка + ≥ 3 разных категорий | `Выбери 3 категории — вернёшь ~{N} ₽ в месяц` | Выбрать → | ProductsTab |

#### Сбережения
| ID | Триггер | Текст баннера | CTA | Действие |
|----|---------|---------------|-----|---------|
| `savings_account` | balance > 30 000, нет накопительного | `{X} ₽ без дела · накопительный даст +{N} ₽/мес` | Открыть → | ProductsTab |
| `deposit_3m` | balance > 80 000, нет вклада | `Открой вклад на 3 месяца → +{N} ₽ сверху` | Открыть → | ProductsTab |
| `deposit_1y` | balance > 200 000, нет вклада | `{X} ₽ в вклад на год → +{N} ₽ к концу срока` | Открыть → | ProductsTab |
| `round_up` | много мелких трат (< 500 ₽ / > 10 шт за мес) | `Округляй покупки → копи незаметно, ~{N} ₽/мес` | Подключить → | ProductsTab |

#### Инвестиции
| ID | Триггер | Текст баннера | CTA | Действие |
|----|---------|---------------|-----|---------|
| `iis` | возраст 25–45, нет ИИС, остаток > 50 000 | `Открой ИИС → вернёшь до 52 000 ₽ из налогов в год` | Открыть → | ProductsTab |
| `portfolio` | сегмент investor, есть инвест. счёт | `Портфель: авто-ребалансировка раз в квартал` | Посмотреть → | ProductsTab |

#### Сервисы
| ID | Триггер | Текст баннера | CTA | Действие |
|----|---------|---------------|-----|---------|
| `prime` | ≥ 5 переводов за месяц | `{N} переводов за месяц · с Прайм они бесплатные` | Подробнее → | ProductsTab |
| `autopay_setup` | недавний просроченный autopayment | `Подключи автоплатёж {Имя} — больше не забудешь` | Подключить → | ProductsTab |
| `insurance_travel` | транзакция в категории Отели / Такси в аэропорт | `Планируешь поездку? Страховка от 300 ₽` | Оформить → | ProductsTab |

### Логика выбора (приоритет сверху вниз)

```
1. autopay_setup   — если в scheduled_payments есть status='Overdue'
2. savings_account — если balance > 30 000 и нет накопительного
3. deposit_*       — если balance > 80 000 и нет вклада
4. cashback_*      — по топ-категории трат за 30 дней
5. iis             — если 25–45 лет, нет ИИС, остаток > 50 000
6. prime           — если ≥ 5 переводов за месяц
7. insurance_travel — если travel-транзакция за последние 7 дней
8. round_up        — fallback если нет других триггеров
```

**Правила исключения:**
- Не показывать продукт из `active_products` пользователя (уже есть)
- Один и тот же оффер — не чаще 1 раза в 3 дня (хранится `last_shown_vygoda` в SharedPreferences)
- Если пользователь закрыл баннер — скрыть этот продукт на 7 дней
- За одну сессию — не более 1 баннера ВЫГОДА

### Расчёт числовой выгоды

| Продукт | Формула |
|---------|---------|
| Cashback | `spend_in_category * cashback_rate` (1–5% в зависимости от категории) |
| Накопительный счёт | `balance * 0.08 / 12` (8% годовых / 12 месяцев) |
| Вклад 3 мес | `amount * 0.18 / 4` (18% годовых / 4 квартала) |
| Вклад 1 год | `amount * 0.16` (16% годовых) |
| Round-up | `avg_transaction_count_per_month * avg_round_amount` (≈ 30–50 ₽/опер.) |

### Данные для демо

| Персона | Показывается | Почему |
|---------|-------------|--------|
| Витя | `cashback_cafe` / `cashback_multi` | много мелких трат, нет кэшбэка |
| Ольга | `autopay_setup` (после оплаты ТТК) → `savings_account` | просрочка ТТК, остаток на дебетовом |
| Артём | `iis` | 29 лет, нет ИИС, есть остаток |

### Баннер в InputActivity

```
┌─ ВЫГОДА ───────────────────────────────────────────────┐
│  Выбери категории — вернёшь ~600 ₽ в месяц  Подробнее →│
└────────────────────────────────────────────────────────┘
```

## Тип 3 — Аналитика 📋 (спроектирована, не реализована)

Подсказка ценна сама по себе. Пользователь получает знание — без призыва что-то купить.

| Триггер | Текст | Действие |
|---------|-------|---------|
| Конец месяца (28–31) | `Апрель закрыт — посмотреть куда ушло?` | → История транзакций |
| Конец месяца + аномалия | `Апрель: потратил на 14 200 ₽ больше, чем в марте` | → История |
| Крупная трата < 24 ч | `Потратил 8 400 ₽ — осталось 23% бюджета на неделю` | → Аналитика |

Баннера нет — Аналитика только в виджете, никакого баннера в InputActivity.

## API — GET /hint

Endpoint возвращает одну персонализированную подсказку для пользователя.

```
GET /hint?user_id=vitya
Authorization: Bearer <app_token>
```

**Response (`HintResponse`):**

| Поле | Тип | Описание |
|------|-----|----------|
| `type` | `"reminder"` / `"vygoda"` / `"none"` | Тип подсказки |
| `payment_id` | string? | ID планового платежа (для reminder) |
| `name` | string? | Название платежа/оффера |
| `amount` | float? | Сумма платежа |
| `days_until_due` | int? | Дней до платежа |
| `is_overdue` | bool? | Просрочен ли платёж |
| `urgency` | string? | `"urgent"` / `"upcoming"` |
| `label` | string? | Текст для виджета (короткий) |
| `offer_id` | string? | ID оффера (для vygoda) |
| `offer_text` | string? | Текст баннера ВЫГОДА |
| `offer_cta` | string? | Текст кнопки CTA |
| `offer_action` | string? | Действие при нажатии |

**Логика приоритетов:**

```
1. hint_override для user_id (установлена через /dashboard/hints)
   ├── type="none"     → HintResponse(type="none")
   ├── type="reminder" → ближайший reminder (или forced_payment_id)
   ├── type="vygoda"   → лучший оффер (или forced_offer_id)
   ├── type="neutral"  → HintResponse(type="none")
   └── type="auto"     → см. п.2 с учётом reminder_enabled/vygoda_enabled
2. Автоматическая логика (без override):
   ├── reminder_enabled → ближайший scheduled_payment (credit_card/loan/autopayment)
   └── vygoda_enabled   → лучший оффер по сегменту/балансу/тратам
3. Ничего не подошло → HintResponse(type="none")
```

## Dashboard управления подсказками

**URL:** `https://vtb.vibefounder.ru/dashboard/hints` (Basic Auth)

Позволяет продуктовому менеджеру:
- Просмотреть текущую подсказку каждого пользователя
- Задать тип подсказки: `auto` / `reminder` / `vygoda` / `neutral` / `none`
- Форсировать конкретный оффер (`forced_offer_id`) или платёж (`forced_payment_id`)
- Переопределить текст, CTA, действие
- Удалить override (вернуть к автоматической логике)

API: `PUT /dashboard/hints/api/{user_id}` (создать/обновить) / `DELETE` (удалить).

## Приоритет между типами

```
При каждом открытии InputActivity выбирается один баннер:

  Напоминание (если есть актуальный платёж)
      ↓ нет
  ВЫГОДА (по логике выше, с учётом ограничений частоты)
      ↓ нет
  ничего (поле ввода чистое)

Нейтральные и Аналитика — только в виджете, в InputActivity не дублируются.
```


## Offline fallback

```
Запрос к /hint (app_token, без PIN)
  ├── ответ: type="reminder" → ReminderBanner
  ├── ответ: type="vygoda"   → VygodaBanner
  ├── ответ: type="none"     → чистое поле ввода
  └── ошибка / timeout       → чистое поле ввода

Нейтральная фраза на виджете — всегда из локального пула NeutralHints.kt.
```

> Ранее при offline/none показывался хардкод VygodaBanner. Теперь — чистое поле ввода, что снижает информационный шум.

## Источники данных

| Сигнал | Источник в прототипе |
|--------|---------------------|
| Время / день / праздник | `System.currentTimeMillis()` + `Calendar` |
| Пол, день рождения | `Persona.gender`, `Persona.birthday` (MainActivity.kt) |
| Плановые платежи | SQLite `scheduled_payments.payment_type` |
| Баланс счёта | SQLite `accounts.balance` |
| История трат | SQLite `transactions` |
| Продукты в наличии | SQLite `users.active_products` (JSON массив) |
| Частота переводов | COUNT от `transactions WHERE type='Transfer'` |
| Последний показанный hint | `SharedPreferences vita_hints` |
| Override подсказок | SQLite `hint_overrides` (управляется через /dashboard/hints) |
| Офферы ВЫГОДА | `db.get_available_offers()` — расчёт на лету по сегменту/балансу/тратам |

## Что не работает и почему

| Фраза / паттерн | Проблема |
|----------------|----------|
| «Снова перевести Маше?» | Навязчиво — пользователь сам помнит |
| «Пятничный перевод?» | Пустая фраза, нет полезной информации |
| «Зарплата пришла — куда отложить?» | Абстрактно, нет персональной цифры |
| «Деньги лежат без дела — вклад 18%?» | Звучит как реклама, нет персональной суммы |
| «Возвращай деньги с каждой покупки» | Слоган, не совет |
| Показывать ВЫГОДА без числа | Нарушает главный принцип |
