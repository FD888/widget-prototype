# VTB Vita — Персонализация и нативная реклама виджета

> Статус: **Draft** — механика прорабатывается, к реализации не приступали  
> Последнее обновление: 12 апреля 2026

---

## Концепция

Виджет показывает персонализированные подсказки — короткие фразы рядом с полем ввода.  
Подсказки делятся на четыре типа:

- **Нейтральные** — простые приятные фразы без информационной нагрузки, появляются чаще всего
- **Напоминания** — своевременные уведомления о предстоящих платежах и событиях
- **Аналитика** — полезное наблюдение на основе данных пользователя, без призыва купить что-то
- **ВЫГОДА** — конкретное предложение продукта ВТБ с явной числовой выгодой

**Ключевой принцип:** виджет не должен создавать информационное давление. Большую часть времени он должен просто быть — нейтрально и ненавязчиво.

**Тест фразы:** звучит ли это как совет от друга-финансиста? Если звучит как менеджер банка — выбросить.

---

## Тип 0 — Нейтральные (ambient)

Фразы без информационного посыла. Их задача — сделать виджет живым, не давить.  
Появляются по умолчанию, когда ни один другой триггер не сработал — и это нормально.

Фразы не кликабельны, не ведут никуда. Просто текст.

**Примеры:**

```
Хорошего дня
Как дела?
Чем могу помочь?
Всё под контролем
Готов помочь
На связи
```

**Правила подбора:**
- Максимально короткие (2–4 слова)
- Без восклицательных знаков — не надо изображать энтузиазм
- Без вопросов, требующих ответа («Как дела?» — риторически, не буквально)
- Ротация случайная, но без повторов два раза подряд

## Тип 1 — Напоминания

Своевременные, конкретные, полезные. Появляются только когда есть реальное событие.  
Не реклама — это сервис.

| Триггер | Фраза | Действие |
|---------|-------|----------|
| За 3 дня до платежа по кредитке | «Через 3 дня платёж по карте — 4 200 ₽» | → экран платежа |
| За 1 день | «Завтра платёж по кредитке — не забудь» | → экран платежа |
| День платежа | «Сегодня платёж по кредитке — 4 200 ₽» | → экран платежа |
| За 3 дня до истечения вклада | «Вклад заканчивается через 3 дня — продлить?» | → вклад |
| Регулярный платёж (ЖКХ, подписка) в этот период | «Обычно в это время оплачиваешь ЖКХ» | → оплата услуг |
| Баланс упал ниже порога (< 1 000 ₽) | «На карте осталось 840 ₽» | — (без действия) |

**Ключевое отличие от ВЫГОДА:** напоминание — это про деньги пользователя, не про продукт банка.  
Его можно показывать без ограничений по частоте, если событие актуально.

## Источники данных для персонализации

Все данные в прототипе — mock. В реальном ВТБ — требуется согласие на маркетинговую обработку (уже собирается при открытии счёта).

| Сигнал | Источник в прототипе | Требует согласия? |
|--------|---------------------|-------------------|
| Время / день недели | `System.currentTimeMillis()` | Нет |
| История операций (суммы, получатели, категории) | Mock JSON | В реальности — да |
| Последнее действие в виджете | `SharedPreferences` (локально) | Нет |
| Профиль пользователя (сегмент, продукты) | Статичный mock | В реальности — да |
| Геолокация | **Не используется** | — |

> 152-ФЗ: поведенческие данные хранятся только локально на устройстве. Кросс-сейл на основе профиля — категория «информирование об услугах банка», не требует отдельного согласия по банковской практике.

## Тип 1 — Аналитика

Подсказка ценна сама по себе. Пользователь получает знание, которого у него не было.  
Никакого продукта не продаётся.

| Триггер | Фраза | Действие по нажатию |
|---------|-------|---------------------|
| Конец месяца (28–31 число) | «Ноябрь закрыт — посмотреть куда ушло?» | → экран аналитики трат |
| Конец месяца + аномалия | «Ноябрь: потратил на 14 200 ₽ больше, чем в октябре» | → экран аналитики трат |
| Крупная трата в последние 24 ч | «Потратил 8 400 ₽ — осталось 23% бюджета на неделю» | → аналитика / бюджет |

**Почему это работает:** конкретная цифра-сюрприз, которую пользователь не видел без банка.

---

## Тип 2 — ВЫГОДА

Формат: цветной тег **`ВЫГОДА`** (как в основном приложении ВТБ) + **числовая выгода** + **1 понятное действие**.

Без тройки `тег + цифра + действие` — не показывать.

| Сигнал | Фраза | Конкретика |
|--------|-------|------------|
| Нет кешбэка, >5 операций/нед | `ВЫГОДА · Выбери 3 категории — вернёшь ~600 ₽ в месяц` | Сумма считается по mock-истории |
| Нет кешбэка, большие траты в кафе | `ВЫГОДА · Кешбэк на кафе вернёт ~400 ₽ с прошлого месяца` | Считается от реальных трат |
| Частые крупные переводы (>50k/нед) | `ВЫГОДА · Страховка переводов — 99 ₽/мес, покрытие до 300 000 ₽` | Цена и покрытие явно |
| Остаток >80k, нет вклада | `ВЫГОДА · 80 000 ₽ на накопительном дадут +1 200 ₽ в месяц` | Считается от реального остатка |
| Сегмент «инвестор», утро раб. дня | `ВЫГОДА · Портфель вырос на 2,3% за неделю — посмотреть?` | Цифра из mock-портфеля |

### Продукты ВТБ в кросс-сейле (приоритет для прототипа)

1. Кешбэк / программа лояльности
2. Накопительный счёт / вклад
3. Страховка переводов
4. Инвестиции

## Что не работает и почему

Список отвергнутых паттернов — чтобы не возвращаться к ним.

| Фраза | Проблема |
|-------|----------|
| «Пятничный перевод?» | Пользователь сам знает когда переводить. Пустая фраза |
| «Зарплата пришла — куда отложить?» | Абстрактно, нет конкретики |
| «Обед — заплатить за кофе?» | Непонятно что предлагается. Бессмысленно |
| «Снова перевести Маше?» | Навязчиво. Пользователь сам помнит кому переводит |
| «Обычно переводишь по пятницам…» | То же самое |
| «Деньги лежат без дела — вклад 18%?» | Звучит как реклама. Нет персональной цифры |
| «Возвращай деньги с каждой покупки» | Слоган, не совет |

## UX механика показа

### Виджет

```
┌─────────────────────────────────────────────┐
│  [ВЫГОДА]  Выбери категории —               │
│            вернёшь ~600 ₽ в месяц          │
│  ___________________________________________│
│  Скажи что сделать...               [🎤]   │
└─────────────────────────────────────────────┘
```

Подсказка располагается над полем ввода. Для аналитических подсказок — без тега.

### InputActivity (после тапа на поле ввода)

Если подсказка была типа ВЫГОДА — в InputActivity всплывает контекстный баннер:

```
┌─────────────────────────────────────────────┐
│ ┌ ВЫГОДА ──────────────────────────────── ┐ │
│ │  Кешбэк на кафе вернёт ~400 ₽          │ │
│ │  с прошлого месяца        [Выбрать →]  │ │
│ └────────────────────────────────────────┘ │
│                                             │
│  Скажи что сделать...               [🎤]   │
└─────────────────────────────────────────────┘
```

Для аналитических подсказок — баннера нет, подсказка становится контекстом / автозаполнением.

**Логика перехода:**
```
Виджет (подсказка видна)
  → пользователь нажимает на поле ввода
  → InputActivity открывается
  → если тип ВЫГОДА: баннер всплывает вверху
  → пользователь нажимает [Выбрать →]
  → deep link в основное приложение ВТБ на оффер
```

## Приоритеты и частотные ограничения

```
Порядок приоритетов:
  1. Аналитика с аномалией (конец месяца + неожиданная цифра)
  2. ВЫГОДА с высокой персональной суммой
  3. Аналитика без аномалии
  4. ВЫГОДА с низкой персональной суммой

Частотные ограничения:
  - Один и тот же ВЫГОДА-оффер — не чаще 1 раза в 3 дня
  - Если пользователь закрыл баннер — скрыть этот продукт на 7 дней
  - За одну сессию — не более 1 ВЫГОДА-подсказки
  - Аналитика — без ограничений, если триггер актуален

Если ни один триггер не подходит:
  → показывать пустой виджет без подсказки (это нормально)
```

## Бизнес-логика генерации подсказки (псевдокод)

```kotlin
fun generateHint(profile: UserProfile, history: List<Transaction>, time: LocalDateTime): Hint? {

    // 1. Аналитика — приоритет, всегда уместна
    if (isEndOfMonth(time) && history.isNotEmpty()) {
        val delta = spendingDeltaVsLastMonth(history)
        return if (delta != null)
            AnalyticsHint("Месяц закрыт: потратил на ${delta} ₽ больше, чем в прошлом — посмотреть?")
        else
            AnalyticsHint("Месяц закрыт — посмотреть куда ушло?")
    }

    val recentBigExpense = history.lastOrNull { it.amount > 5000 && it.hoursAgo < 24 }
    if (recentBigExpense != null) {
        val budgetLeft = budgetRemainingPercent(history)
        return AnalyticsHint("Потратил ${recentBigExpense.amount} ₽ — осталось ${budgetLeft}% бюджета на неделю")
    }

    // 2. ВЫГОДА — только если есть конкретная числовая выгода
    if (!profile.hasCashback) {
        val saving = estimateMonthlyCashback(history)
        if (saving > 100)
            return VygodaHint("Выбери категории — вернёшь ~${saving} ₽ в месяц", product = CASHBACK)
    }

    val balance = profile.mainAccountBalance
    if (balance > 50_000 && !profile.hasDeposit) {
        val income = estimateMonthlyDepositIncome(balance)
        return VygodaHint("${balance.format()} ₽ на накопительном дадут +${income} ₽ в месяц", product = DEPOSIT)
    }

    val bigTransfers = history.filter { it.type == TRANSFER && it.amount > 30_000 }
    if (bigTransfers.size >= 2 && !profile.hasTransferInsurance)
        return VygodaHint("Страховка переводов — 99 ₽/мес, покрытие до 300 000 ₽", product = INSURANCE)

    // 3. Ничего подходящего — не показывать ничего
    return null
}
```

## Метрики для оценки

| Метрика | Что измеряет | Цель |
|---------|-------------|------|
| Hint shown rate | % сессий, где подсказка показана | Понять насколько часто триггеры срабатывают |
| Hint CTR | % переходов «виджет → InputActivity» через подсказку | Насколько подсказка мотивирует к действию |
| Banner CTR | % нажатий на ВЫГОДА-баннер в InputActivity | Конверсия интереса в клик |
| Dismiss rate | % закрытий баннера | Сигнал что оффер нерелевантен |
| ВЫГОДА / сессия | Среднее число ВЫГОДА-показов на сессию | Должно быть ≤ 1 |
