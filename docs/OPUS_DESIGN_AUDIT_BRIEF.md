# Opus Design Audit Brief — VTB Vita

> **Для кого:** Opus 4.7 как дизайн-аналитик  
> **Твоя задача:** Сравнить текущий UI виджета VTB Vita с реальным приложением ВТБ Онлайн, выявить визуальные несоответствия и написать детальный план исправлений в `docs/DESIGN_IMPROVEMENTS.md`.  
> **Что НЕ делать:** не писать код, не предлагать архитектурные изменения, только дизайн-экспертиза.

---

## Контекст проекта

**VTB Vita** — прототип голосового/текстового виджета для Android. Пользователь говорит одну фразу («переведи 500 Вите») → виджет выполняет операцию без открытия приложения.

Прототип создан для финальной защиты перед жюри ВТБ. Он должен выглядеть как продакшн-версия ВТБ Онлайн, но сейчас дизайн-элементы формально соответствуют дизайн-системе Omega, а **общая картинка не соответствует** — детали правильные, конструктор собран неправильно.

**Дизайн-система:** Omega (ВТБ). Токены уже внедрены в код (`OmegaColor.kt`, `OmegaType.kt`, `OmegaDimensions.kt`). Проблема — в логике использования, иерархии, композиции.

## Шаги работы

### Шаг 1 — Прочитать все скриншоты

Прочитай все файлы из обеих групп с помощью Read tool. Просматривай попарно: сначала эталон ВТБ → потом аналогичный экран Виты.

Ключевые пары для сравнения:
- ВТБ главная ↔ Вита главная
- ВТБ перевод (все этапы) ↔ Вита настройка перевода
- ВТБ чат/ассистент ↔ Вита input activity + чат
- ВТБ пополнение ↔ Вита пополнение

### Шаг 2 — Изучить текущие файлы темы

Прочитай для понимания токенов:
- `android/app/src/main/java/com/vtbvita/widget/ui/theme/OmegaColor.kt` — цвета
- `android/app/src/main/java/com/vtbvita/widget/ui/theme/OmegaType.kt` — типографика
- `android/app/src/main/java/com/vtbvita/widget/ui/theme/OmegaDimensions.kt` — spacing/radius
- `android/app/src/main/java/com/vtbvita/widget/ui/components/OmegaComponents.kt` — компоненты

Это нужно чтобы понять **что уже есть** в системе, и при рекомендациях ссылаться на правильные токены, а не придумывать новые.

### Шаг 3 — Анализ и написание плана

По каждому экрану/зоне — детальный разбор расхождений с эталоном. Пиши честно и конкретно.

### Шаг 4 — Сохранить результат

Записать в `docs/DESIGN_IMPROVEMENTS.md`.

## Дизайн-система Omega — ключевой контекст

Токены уже используются в проекте. При рекомендациях **ссылайся на конкретные токены** из `OmegaColor.kt`, `OmegaType.kt`, `OmegaDimensions.kt` — не придумывай hex-коды.

### Ключевые принципы дизайна ВТБ Онлайн (из анализа скриншотов)

1. **Тёмная тема — базовая.** Фон `#1E2024` (spaceGraphite/600), поверхности `#22252B` (titanGray/900).
2. **Синий brand цвет** — `#0160EC` (omegaBlue/600). Используется точечно: CTA-кнопки, акценты, активные иконки.
3. **Карточки с глубиной** — не плоские блоки, а layered поверхности с тонкими тенями и правильным border-radius.
4. **Типографика с иерархией** — чёткое разделение Display/Headline/Body по размеру и весу.
5. **Цвет как семантика** — зелёный только для успеха/прибыли, красный для ошибки/расхода. Не как «декор».
6. **Bottom sheet как основной паттерн** для дополнительных действий.
7. **Иконки из Omega** — одинаковый стиль, одинаковый размер (24dp), правильный цвет.
8. **Контакты с аватарами** — инициалы в круглом контейнере с правильными цветами фона.

### Правила написания

1. **Начинай каждый экран с имени Activity-файла** в скобках — это то, что будет реализовывать разработчик.
2. **Называй конкретные токены** из кода (`OmegaColor.BrandPrimary`, `OmegaType.HeadlineL`, `OmegaDimensions.RadiusMd`) — не абстрактные описания.
3. **Приоритет проблем:** HIGH / MEDIUM / LOW в начале каждого блока.
4. **Не пиши код** — только визуальные инструкции ("использовать Surface цвет вместо фонового", "уменьшить border-radius с pill до 16dp").
5. **Ссылки на файлы** — только конкретные `.kt` файлы из `android/app/src/main/java/com/vtbvita/widget/`.
6. **Системные проблемы** (цвет, типографика, spacing) — вынеси в отдельный раздел "Системные правки", не повторяй в каждом экране.

## Известные кандидаты на проблемы (из первичного осмотра)

Эти наблюдения — отправная точка, не ограничение. Ты сам увидишь больше.

- Просмотр баланса выглядит не так как в реальном приложении. Я бы хотел, чтобы возможно была как то показана карточка, и баланс, наглядно. А не безллико, с текстом.
- Нет экрана подтверждения при переводе, возможно не тот UX флоу при переводе и пополнении номера
- Нет возможности перевести в другие страны, и выбрать другие банки (иконка выбора "другой" есть, но там не тот значек)
- **Bottom navigation** — иконки и их стиль нужно проверить vs эталон
- **Input Activity** — синий gradient pill с микрофоном выглядит нестандартно vs ВТБ-чат паттерн
- **Чипы быстрых сумм** (+500₽, +1000₽) — стиль и spacing
- **PIN-экран** — проверить соответствие Omega стилю. Он как будто должен быть на темном фоне.
- **Bottom sheet** перевода — отступы, тени, handle bar
- При записи голосом иконки не те
- Иконка "назад" не как в приложении на этапах настройки перевода
- Нет линий и обводок каких то.
