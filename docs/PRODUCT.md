## Суть продукта

Голосовой/текстовый виджет на главном экране Android.
Пользователь говорит или пишет команду → биометрия/пин → подтверждает в модале → операция выполнена.
Без открытия приложения. Цель: 8 секунд вместо 60.

**Целевая аудитория:** 25–45 лет, зарплатные клиенты ВТБ, 2–5 операций в день.

---

## Три банковские операции MVP (всё что нужно для прототипа)

| Intent | Пример фразы | Что происходит |
|--------|-------------|---------------|
| `transfer` | «Переведи Кате тысячу» | NLP → получатель из истории → модал → транзакция |
| `balance` | «Покажи баланс» | Запрос баланса → показ в виджете |
| `topup` | «Положи 300 на мой телефон» | Определяет номер → модал → пополнение |

По возможности добавить перекидывание в приложение (можно в MVP приложении симулировать фронтенд банковского приложения, чтобы было место, в которое виджет демонстративно может отправить пользователя после соответствующего запроса)

---

## Флоу основной операции

```
Команда (голос/текст)
    → STT (только для голоса, в прототипе Android SpeechRecognizer или skip)
    → NLP: intent + сущности (получатель, сумма, номер)
    → Биометрия / PIN 
    → Модал подтверждения (получатель, сумма, баланс после)
    → [Отправить] → статус в виджете
```

---

## Состояния виджета

**Активная сессия** — показывает поле ввода + контекстную полосу.
**Нет сессии** — только кнопка «Войти в ВТБ Онлайн». Никаких данных.

> В прототипе: сессия имитируется — виджет всегда в активном состоянии для демо.

---

## Модал подтверждения

```
┌─────────────────────────────┐
│  Катя Иванова  •4521        │
│  1 000 ₽  с основного счёта │
│  Баланс после: 14 320 ₽     │
│                             │
│  [ Отмена ]  [ Отправить ]  │
└─────────────────────────────┘
```

NLP может ошибиться → модал обязателен как защита.
Если несколько кандидатов — показываем список для выбора.

---

## Контекстная полоса (нижняя строка виджета)

Приоритет сообщений:
1. Статус последней операции (60 сек)
2. Срочные события банка (списание сегодня/завтра)
3. Информационные события (курс, кэшбэк)

> В прототипе Phase 1: статичный mock-текст. Phase 2 (C-06): динамика.

---

## Переход в основное приложение

Если запрос пользователя совпадает с интеграционными сценариями бота ВТБ Онлайн, либо точно понятен раздел, то:
1) Виджет может перебросить в приложение в нужное место (к примеру, в раздел со вкладом)
2) Продолжить диалог в боте (в контексте первого запроса)

Пример: "Открой вклады" - виджет открывает основное приложение - авторизация - открывается сразу вкладка вклады.

> В прототипе Phase 1: статичный mock-текст. Phase 2 (C-06): динамика.

## Граничные случаи (реализовать в Phase 2)

| Ситуация | Поведение |
|----------|-----------|
| NLP не распознал | Модал с полем ручного ввода |
| Несколько кандидатов | Список на выбор |
| Нет интернета | Поле пустое, метка «Нет сети» |
| Биометрия провалилась | 3 попытки → PIN → блок |

---

## Что виджет НЕ делает (out of scope)

- Логин без основного приложения
- Хранение финансовых данных локально
- Работа без установленного VTB Online (в прототипе это игнорируется)
