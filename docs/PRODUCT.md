> Этот документ — выжимка для команды разработки.
> Полные документы: `../Документы проекта/Гипотеза виджет.md` и `VTB Vita — Product Logic.md`

---

## Суть продукта

Голосовой/текстовый виджет на главном экране Android.
Пользователь говорит или пишет команду → модал подтверждения → операция выполнена.
Без открытия приложения. Цель: 8 секунд вместо 60.

**Целевая аудитория:** 25–45 лет, зарплатные клиенты ВТБ, 2–5 операций в день.

---

## Банковские операции MVP

| Intent | Пример фразы | Что происходит |
|--------|-------------|---------------|
| `transfer` | «Переведи Кате тысячу» | Выбор контакта → сумма / банк / счёт → подтверждение → статус |
| `balance` | «Покажи баланс» | Mock API → баланс в BottomSheet |
| `topup` | «Положи 300 на мой телефон» | Номер + оператор → подтверждение → статус |

---

## Расширенные возможности (реализовано)

| Тип | Примеры фраз | Механизм |
|-----|-------------|---------|
| Будильник | «Разбуди в 7:30» | `AlarmClock.ACTION_SET_ALARM` |
| Таймер | «Таймер на 10 минут» | `AlarmClock.ACTION_SET_TIMER` |
| Звонок | «Позвони маме» | `ACTION_DIAL` + контакты |
| Запуск ПО | «Открой Telegram» | `getLaunchIntentForPackage` |

> Это демонстрирует потенциал виджета как универсального ассистента, не ограниченного банкингом.

## Флоу основной операции

```
Команда (текст / голос)
    → SystemIntentHandler: системный интент? → запускаем стандартное приложение
    → Если нет: NLP (Mock API /parse) → intent + сущности
    → Модал подтверждения (получатель, сумма, баланс после)
    → [Отправить] → /confirm → статус в виджете (10 сек)
```

---

## Состояния виджета

**Активная сессия** — промпт «Как настроение, Денис?» (из Persona.widgetPrompt).
Нажатие на промпт → InputActivity в режиме TEXT.
Нажатие на микрофон → InputActivity в режиме RECORDING.

**Нет сессии** — текст «Войдите в аккаунт», оба элемента ведут в MainActivity.
При выходе из профиля виджет немедленно переключается в неактивное состояние.

---

## Флоу авторизации (прототип)

```
MainActivity (выбор профиля)
    → PinEntryActivity (4-значный PIN)
    → SessionManager.login()
    → VitaWidgetProvider.defaultViews() → персонализированный промпт
    → MockBankActivity (скриншоты + таббар)
        → кнопка выхода → SessionManager.logout() → виджет «Войдите»
```

Персоны:
- **Денис** (активен, PIN 1234) — зарплатный клиент, промпт «Как настроение, Денис?»
- Маша, Яна — «скоро», недоступны

## Модал подтверждения

```
┌─────────────────────────────┐
│  Катя Иванова  •4521        │
│  1 000 ₽  с основного счёта │
│  Баланс после: 14 320 ₽     │
│                             │
│  [ Отмена ]  [ Отправить ]  │
└─────────────────────────────┘
```

NLP может ошибиться → модал обязателен как защита от случайных операций.

---

## Голосовой ввод (реализовано)

- MediaRecorder → запись в .3gp
- Polling maxAmplitude каждые 80 мс → анимация waveform (15 столбиков)
- Счётчик времени записи (мм:сс)
- Распознавание (STT) — Phase 2

---

## Переход в основное приложение

Виджет открывает MockBankActivity (симуляция банка).
В реальном продукте: deeplink в нужный раздел VTB Online.

---

## Граничные случаи (Phase 2)

| Ситуация | Поведение |
|----------|-----------|
| NLP не распознал | Показать ошибку, предложить уточнить |
| Несколько кандидатов | Список на выбор в ContactPicker |
| Нет интернета | Toast «Нет сети», чипы остаются активны |
| Mock API недоступен | Graceful fallback, isLoading = false |

---

## Что виджет НЕ делает (out of scope прототипа)

- Реальные банковские транзакции
- Настоящая биометрия
- Хранение финансовых данных локально
- Логин без вспомогательного экрана выбора профиля
