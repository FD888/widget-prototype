# Задача: Разрешение контактов при переводе

**Статус:** Не начата  
**Приоритет:** Высокий — нужна до финальной защиты 19 апреля  
**Сложность:** ~1 рабочий день

---

## Контекст

Сейчас при команде "переведи маше 500" NLP-парсер возвращает `recipient: "маше"`, и сервер ищет это имя по точному совпадению в `MOCK_CONTACTS`. Это ломается при:
- склонениях ("маше" ≠ "маша")
- фамилиях ("паше коноплеву")
- неоднозначных именах (несколько "Паш" в контактах)

Нужно переделать флоу так, чтобы поиск и сопоставление шли через **реальную книгу контактов на телефоне**.

---

```
  │  Павел К.                       │  ← mock банковского имени (см. ниже)
  │                                 │
  │  Сумма: 100 ₽                   │  ← из команды
  └─────────────────────────────────┘
```

## Архитектурные решения

### Где живут данные

| Данные | Где хранится | Кто читает |
|--------|-------------|-----------|
| Контакты (имя + телефон) | Книга телефона (ContactsContract) | Android (локально) |
| Банковское имя получателя | Mock на сервере (MOCK_CONTACTS по номеру) | `/command` endpoint |
| Сумма | Парсится NLP (L1/L2) | Android → сервер |

**Контакты на сервер НЕ загружаем** — приватные данные, не нужно.

### Банковское имя получателя (mock)

Реальный банк отдаёт маскированное имя по номеру телефона.  
В прототипе: берём имя из книги контактов → форматируем как `"Имя Ф."`:

```
"Паша Коноплев СПБГУ"  →  "Паша К."
"мама" (нет фамилии)   →  "Светлана В."  ← hardcode для демо-контактов
```

Логика: `firstName + " " + lastName[0] + "."` — если есть фамилия.  
Если фамилии нет или имя одно слово — fallback на `"Клиент ВТБ"`.

## Что нужно реализовать

### 1. `ContactMatcher.kt` (новый файл)

Локальный нечёткий поиск по ContactsContract.

```
Вход:  recipient_raw = "паше коноплеву"
Выход: List<ContactCandidate> отсортированный по score

ContactCandidate:
  - displayName: String     ("Паша Коноплев СПБГУ")
  - phone: String           ("+79181111111")
  - bankDisplayName: String ("Паша К.")  ← генерируем локально
  - score: Float            (0.0–1.0)
```

**Алгоритм:**
1. Нормализация: lowercase, ё→е, стрипаем склоняемые окончания
   ```
   "паше коноплеву" → ["паш", "коноплев"]
   "маше"           → ["маш"]
   "яне"            → ["ян"]
   ```
   Суффиксы для strip (дательный, родительный, винительный): `-е, -у, -ю, -а, -я, -ы, -и, -ом, -ем, -ой, -ей, -ого, -его, -ому, -ему`

2. Запрос в ContentProvider по каждому токену (LIKE '%токен%')

3. Scoring кандидатов:
   - +0.5 за совпадение имени (первый токен)
   - +0.4 за совпадение фамилии (второй токен если есть)
   - +0.1 за совпадение любого другого слова
   - Кандидат включается если score ≥ 0.4

4. Сортировка по score desc

**Порог однозначности:** score[0] ≥ 0.8 И score[0] − score[1] ≥ 0.3 → считаем однозначным.  
Иначе → показываем список.

### 2. `ContactPickerBottomSheet.kt` (новый файл)

Bottom sheet с кандидатами. Показывается поверх InputActivity когда кандидатов 2+.

```
Layout:
  TextView "Кому переводим?"
  RecyclerView с кандидатами:
    каждая строка: displayName + телефон (маскированный: +7 (918) ***-**-11)
  Кнопка "Другой получатель" → открывает ContactPickerActivity
```

При тапе на контакт → закрывается, возвращает выбранный `ContactCandidate`.

### 3. Обновить `InputActivity.kt`

В методе `onTransfer(recipient: String?, amount: Double?)` вместо прямого открытия TransferDetails:

```kotlin
if (recipient != null) {
    val candidates = ContactMatcher.search(recipient, context)
    when {
        candidates.isEmpty() -> {
            // Контакт не найден → ContactPickerActivity с суммой
            openContactPickerWithAmount(amount)
        }
        candidates.size == 1 || candidates[0].isHighConfidence() -> {
            // Однозначно → сразу TransferDetailsActivity
            openTransferDetails(candidates[0], amount)
        }
        else -> {
            // Несколько → bottom sheet выбора
            ContactPickerBottomSheet.show(supportFragmentManager, candidates, amount) { selected ->
                openTransferDetails(selected, amount)
            }
        }
    }
} else {
    openContactPickerWithAmount(amount)
}
```

### 4. Обновить `TransferDetailsActivity.kt`

Добавить отображение:
- Имя из книги контактов (`ContactCandidate.displayName`)
- Телефон (маскированный для показа, полный для отправки)
- Банковское имя (`ContactCandidate.bankDisplayName`) под именем контакта

В `/command` отправлять **номер телефона** вместо имени:
```kotlin
// Было:
CommandRequest(intent = "transfer", recipient = parsedRecipientName, amount = amount)

// Станет:
CommandRequest(intent = "transfer", recipient = selectedContact.phone, amount = amount)
```

### 5. Обновить `MOCK_CONTACTS` на сервере

Сейчас ключи — имена (`"маша"`, `"яна"`). После изменения `/command` будет получать **номер телефона**.  
`_PHONE_INDEX` уже есть в `main.py` — этот путь уже работает.  
Проверить что `_PHONE_INDEX` корректно строится для всех контактов демо.
